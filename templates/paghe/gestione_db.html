Guida — Gestione Database
Cosa sono i profili database

Ogni profilo è un file SQLite indipendente. Il profilo Principale contiene i tuoi dati reali. I profili di prova (Prova Casistiche, Prova Varie, Prova Molti, Prova Singolo Contratto) sono database separati popolati con dati fittizi per testare l'applicazione senza toccare i dati di produzione.

Come creare un database di prova
  1. Nella lista Profili Disponibili, individua il profilo con scritto Non creato.
  2. Clicca Crea Database: apparirà un overlay con spinner.
  3. Attendi il completamento (da pochi secondi a ~30 secondi per profili grandi).
  4. La pagina si ricaricherà automaticamente. Ora il profilo mostra i pulsanti Attiva ed Elimina.
Come attivare un database (cambiare DB attivo)
  1. Clicca Attiva sul profilo desiderato.
  2. Verrai reindirizzato al login del database scelto.
  3. Accedi con le credenziali di quel database (per i DB di prova: admin / admin).
Attenzione: Dopo il cambio, la sessione precedente non è più valida. Devi sempre rifare il login. I dati dell'altro database restano intatti.
Come tornare al database principale

Nella lista Profili Disponibili, clicca Attiva sul profilo Principale. Verrai reindirizzato al login del database principale. Inserisci le tue credenziali normali.

Come eliminare un database di prova
  1. Assicurati che il database da eliminare non sia quello attivo (cambia prima sul principale).
  2. Clicca Elimina: apparirà un messaggio di conferma.
  3. Conferma l'eliminazione. Il file .sqlite3 verrà cancellato definitivamente.
Nota: L'eliminazione è irreversibile. I dati cancellati non possono essere recuperati in alcun modo.
Note importanti
  • Dopo ogni cambio di database devi sempre effettuare nuovamente il login — la sessione è legata al database attivo.
  • Credenziali dei database di prova: admin / admin (utente superadmin con tutti i permessi).
  • Ogni database è completamente isolato: i dati di un profilo non sono visibili dagli altri.
  • Il database Principale non può essere eliminato.
